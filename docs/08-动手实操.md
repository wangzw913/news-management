## 一、第一次运行项目

### 前提条件检查

打开终端（CMD 或 Git Bash），依次输入：

```bash
java -version        # 你需要看到 Java 21（或 17+）
node -v              # 需要看到 v20+
npm -v               # 需要看到 10+
```

如果 Java 版本不是 17+：
- 你的电脑上已经有 JDK 21：`C:\Program Files\Java\jdk-21.0.10`
- 在系统环境变量中设置 `JAVA_HOME=C:\Program Files\Java\jdk-21.0.10`

### 步骤 1：启动 MySQL

WampServer 自带 MySQL。确保 WampServer 托盘图标是绿色的。
如果没装 WampServer，也可以单独启动 MySQL 服务。

### 步骤 2：创建数据库

```bash
# 用 WampServer 自带的 MySQL
"C:\wamp\bin\mysql\mysql5.6.12\bin\mysql" -uroot -e "CREATE DATABASE IF NOT EXISTS news_v3 DEFAULT CHARSET utf8"
```

### 步骤 3：导入建表 SQL

```bash
"C:\wamp\bin\mysql\mysql5.6.12\bin\mysql" -uroot news_v3 < C:\wamp\www\news_v3\news-server\src\main\resources\db\schema.sql
```
或者：项目启动时会自动创建表（DataInitializer.java），但如果 MySQL 版本太低可能有问题，手动导入更稳妥。

### 步骤 4：启动后端

打开终端，进入后端目录：
```bash
cd C:\wamp\www\news_v3\news-server

# 用 Maven Wrapper 启动（不需要单独装 Maven）
mvnw.cmd spring-boot:run
```

看到这行就说明启动成功：
```
Started NewsApplication in 3.713 seconds
```

### 步骤 5：启动前端

另开一个终端，进入前端目录：
```bash
cd C:\wamp\www\news_v3\news-web

# 第一次需要安装依赖
npm install

# 启动开发服务器
npm run dev
```

看到这行就说明启动成功：
```
➜  Local:   http://localhost:5173/
```

### 步骤 6：打开浏览器

访问 `http://localhost:5173`

用 `admin / admin123` 登录，选择"管理员"。（如果登录页有选角色的话——实际上我们已经在最新版去掉了角色选择，直接输入用户名密码即可）

## 二、走一遍完整业务流程

### 流程 1：编辑写稿 → 审核员审核 → 用户看到

```
1. 用 editor / editor123 登录
   → 自动进入编辑仪表盘

2. 点击"写新稿件"
   → 进入写稿页
   → 标题：测试新闻标题
   → 分类：科技
   → 内容：这是一篇测试新闻的正文内容，包含足够多的文字来测试系统功能。
   → 封面：点击上传区域，选一张图片
   → 点击"提交审核"

3. 登出，用 auditor / auditor123 登录
   → 审核队列看到刚才提交的稿件
   → 点击摘要区展开全文进行审核
   → 点击"审核通过"

4. 稿件发布成功
   → 编辑会收到通知（审核通过）

5. 登出，用 user / user123 登录
   → 首页看到这篇新闻（如果置顶则在轮播中）
   → 点进去 → 收藏 + 点赞 + 评论
   → 底部出现相关推荐
```

### 流程 2：退回修改

```
1. editor 提交一篇稿件
2. auditor 审核 → 发现数据不足 → 填写退回原因 → 点击退回
3. editor 登录 → 看到被退回的稿件及原因 → 点击编辑 → 修改内容 → 重新提交
4. auditor 再次审核 → 通过
```

### 流程 3：管理员管理用户

```
1. admin 登录 → 左侧菜单"用户管理"
2. 看到所有用户列表 → 可以启用/停用用户
3. 被停用的用户无法登录（提示"账号已被停用"）
```

### 流程 4：分类管理

```
1. admin 登录 → 左侧菜单"分类管理"
2. 点击新增 → 名称"军事" → 排序 8 → 保存
3. 首页导航栏出现"军事"分类
4. 编辑写稿时可以选择"军事"分类
```

## 三、打断点调试（理解代码执行流程）

### 后端打断点

如果你用 IntelliJ IDEA：
1. 用 IDEA 打开 `C:\wamp\www\news_v3\news-server`
2. 等待 Maven 依赖下载完成
3. 找到 `AuthController.java` → `login` 方法

1. 浏览器打开 `http://localhost:5173`
2. 按 F12 打开开发者工具 → Sources 标签

5. 输入用户名密码点登录 → 断点触发
6. 鼠标悬停在变量上可以看值

## 四、修改代码看效果

### 练习 1：修改首页轮播速度

打开 `UserPortal.vue`，找到：
```html
<el-carousel :interval="4000" height="280px">
```
把 `4000` 改成 `2000`（2 秒切换），保存，浏览器自动刷新，轮播变快了。

### 练习 2：修改每页显示条数

打开后端 `application.yml`，找到：
```yaml
page-size: 10   # 这个可能没有，在 NewsController 默认参数里
```

在 `NewsController.java` 里找到 `@RequestParam(defaultValue = "10") int size`，改成 `"5"`。重启后端，列表每页只显示 5 条了。

### 练习 3：添加一个新的分类图标

打开 `UserPortal.vue`，找到：
```javascript
const icons = { 1:'🏀', 2:'📈', 3:'💻', 4:'📚', 5:'🏛️', 6:'🎓', 7:'🏠' }
```
加一个 `8:'🪖'`。新增一个分类（排序 8），首页卡片会显示🪖图标。

### 练习 4：修改推荐数量

打开 `NewsDetail.vue`，找到：
```javascript
async function loadRelated() {
  try { relatedNews.value = await request.get(`/recommend/similar/${route.params.id}?limit=5`) } catch {}
}
```
把 `limit=5` 改成 `limit=10`，详情页会显示 10 篇推荐。

## 六、常见问题排查

### 后端启动报 "port 8080 already in use"
→ 上次没关干净，任务管理器里找到 java.exe 杀掉，或重启电脑。

### 后端启动报 "Access denied for user 'root'"
→ MySQL 密码不对。检查 application.yml 的 datasource.password。

### 前端报 "Cannot find module 'element-plus'"
→ 依赖没装。在 news-web 目录下运行 `npm install`。

### 登录后页面空白
→ 按 F12 看 Console 报错。大概率是后端没启动或 API 路径不对。

### 注册后不能登录
→ 注册成功后需要手动切换到登录 tab 重新登录。注册不会自动登录。

### 修改代码后不生效
→ 前端：Vite 有 HMR（热更新），保存即生效。后端：需要重启（有 DevTools 的情况下会自动重启）。

## 七、自己部署到服务器（加分项）

如果你有一台云服务器（阿里云/腾讯云学生机很便宜）：

```bash
# 1. 把代码传到服务器
git clone <你的仓库地址>

# 2. 装 Docker（服务器上）
curl -fsSL https://get.docker.com | sh

# 3. 一键启动
cd news_v3
docker compose up -d

# 4. 访问 http://你的服务器IP
```

面试时可以说："项目已部署在云服务器上，可以现场演示。"——这个是绝杀。
